# Pierwsze deployment do Cloudflare Pages

## Jednorazowe ręczne deployment (lokalnie)

```bash
# Zbuduj aplikację
npm run build

# Deploy do Cloudflare Pages (utworzy projekt jeśli nie istnieje)
npx wrangler pages deploy dist --project-name=NAZWA_PROJEKTU

# Wrangler zapyta o:
# - API Token (lub zaloguje Cię przez browser)
# - Account ID
```

Po pierwszym deploymencie projekt będzie istniał i GitHub Actions będzie działać automatycznie.

## Lub utwórz projekt przez Dashboard

1. https://dash.cloudflare.com/
2. Workers & Pages → Create application → Pages → Upload assets
3. Wpisz nazwę projektu
4. Skip upload (lub wrzuć placeholder)
5. Projekt gotowy dla GitHub Actions

## Konfiguracja GitHub Secrets

Po utworzeniu projektu, ustaw w GitHub:

- `CLOUDFLARE_PROJECT_NAME` - nazwa projektu (np. "10x-athletica")
- `CLOUDFLARE_API_TOKEN` - z Cloudflare → My Profile → API Tokens
- `CLOUDFLARE_ACCOUNT_ID` - z Cloudflare dashboard

## Environment Variables w Cloudflare

W projekcie Cloudflare Pages → Settings → Environment variables:

- `SUPABASE_URL`
- `SUPABASE_KEY`
- `OPENROUTER_API_KEY`
